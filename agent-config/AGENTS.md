## 工作区定义

### 📂 主工作区

- **路径**: `{workspace_root_dir}`

## 启动指令

### 🚀 启动时应该做的

1. **读取启动上下文**
   - 读取 `agent-config/MEMORY.md`（如果有）
   - 读取 `agent-config/AGENTS.md`（本文件）
   - 读取 `agent-config/SOUL.md`（人格定义）

2. **检查待办事项**
   - 读取 `agent-config/HEARTBEAT.md`（如果有）
   - 检查 `memory/YYYY-MM-DD.md`（今天的日志）

3. **准备接收任务**
   - 输出欢迎消息
   - 显示当前工作区
   - 等待用户指令

---

## 任务执行流程

### 📋 标准流程

1. **接收任务** - 从用户消息中理解需求
2. **分解任务** - 如果需要，分解为子任务
3. **分配任务** - 使用 `sessions_spawn` 分配给子 Agent
4. **等待结果** - 使用 `sessions_yield()` 等待完成
5. **汇总结果** - 整理并回复用户

### 🔄 循环流程

- 完成任务后，**必须**检查 `TASK_QUEUE.md`（如果有）
- 如果还有待处理项目，自动开始下一个
- 如果没有，进入待命状态

---

## 文件操作规则

### ✅ 应该做的

1. **使用相对路径** - 相对于 `{workspace_root_dir}`
2. **创建备份** - 修改重要文件前，先备份
3. **记录操作** - 在 `memory/YYYY-MM-DD.md` 中记录
4. **使用 UTF-8** - 所有文件使用 UTF-8 编码

### ❌ 不应该做的

1. **不要使用绝对路径** - 避免硬编码路径
2. **不要修改 MEMORY.md 直接** - 通过记忆管理功能
3. **不要删除日志文件** - `memory/` 目录中的文件是历史记录
4. **不要覆盖整个文件** - 总是读取 → 修改 → 写回
