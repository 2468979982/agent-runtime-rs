# HEARTBEAT.md - 心跳任务列表

## 用途说明

这个文件定义了 **心跳检查时应该做的事情**。

**心跳（Heartbeat）**：OpenClaw 定期向 Agent 发送的轮询消息，让 Agent 有机会主动检查事项、执行后台任务。

---

## 📋 心跳任务列表

### ✅ 应该检查的事项

#### 1. **邮件** 📧
- 检查是否有紧急未读邮件
- 如果有，通知用户

#### 2. **日历** 📅
- 检查未来 24-48 小时内的事件
- 如果快到了（<2小时），提醒用户

#### 3. **提及/通知** 🔔
- 检查 Twitter/社交网络通知
- 如果有重要提及，通知用户

#### 4. **天气** 🌤️
- 如果有相关信息（例如：用户可能出门），提供天气信息

#### 5. **任务状态** 📋
- 检查 `TASK_QUEUE.md`（如果有）
- 如果有待处理项目，考虑是否要开始处理

#### 6. **记忆文件** 🧠
- 检查 `memory/YYYY-MM-DD.md`（今天的日志）
- 如果有未完成的任务，考虑是否要继续

#### 7. **心跳状态** 💓
- 检查 `memory/heartbeat-state.json`
- 记录上次检查的时间，避免重复检查

---

## 💓 心跳状态文件格式

**路径**: `memory/heartbeat-state.json`

**内容**:
```json
{
  "lastChecks": {
    "email": 1703275200,
    "calendar": 1703260800,
    "weather": null,
    "tasks": 1703275200
  },
  "lastHeartbeat": 1703275200,
  "heartbeatCount": 42
}
```

## ⏰ 心跳频率建议

### 🔄 可以批量检查的事项

- **邮件 + 日历 + 通知** → 可以在一次心跳中检查多个事项
- **优点**: 减少 API 调用次数
- **缺点**: 如果某一项检查很慢，会延迟其他项

### 🕐 不应该频繁检查的事项

- **天气** → 每小时检查一次就够了
- **任务状态** → 每天检查 2-3 次
- **记忆文件** → 每次心跳都检查（很快）

### 🚀 应该立即检查的事项

- **紧急邮件** → 如果有实时推送，立即检查
- **日历提醒** → 如果快到时间了（<2小时），频繁检查

## 🤔 何时应该回复用户？

### ✅ 应该回复的情况

1. **重要邮件到达**
2. **日历事件快到了**（<2小时）
3. **发现了有趣的东西**（想分享给用户）
4. **已经 >8 小时没说话了**

### ⏳ 应该保持安静的情况（返回 HEARTBEAT_OK）

1. **深夜**（23:00-08:00）除非紧急
2. **用户明显很忙**（从对话历史判断）
3. **自上次检查后没新东西**
4. **刚刚检查过**（<30 分钟）

## 🎯 心跳与 Cron 的区别

| 特性 | 心跳（Heartbeat） | Cron |
|------|------------------|------|
| **时机** | 周期性轮询（例如：每 30 分钟） | 精确时间（例如：每天 09:00） |
| **上下文** | 需要对话上下文（最近消息） | 可以独立运行（isolated session） |
| **适用场景** | 批量检查多个事项 | 单一定时任务 |
| **优点** | 减少 API 调用（批量） | 时间精确 |
| **缺点** | 时间可能漂移（不一定精确 30 分钟） | 如果每个事项都建 cron，会很多 |

### 💡 使用建议

- **使用心跳**：如果需要**批量检查**多个事项（邮件 + 日历 + 通知）
- **使用 cron**：如果**时间精确**很重要（例如："每天 09:00 准时提醒"）

## 📋 如何添加心跳任务？

在 `agent-config/HEARTBEAT.md` 中添加任务：

```markdown
## 📋 定期任务

### 📧 检查邮件
- **频率**: 每 30 分钟
- **条件**: 如果有未读邮件，通知用户
- **状态**: ✅ 启用

### 📅 检查日历
- **频率**: 每 2 小时
- **条件**: 如果未来 24 小时有事件，提醒用户
- **状态**: ✅ 启用

### 🧠 检查任务状态
- **频率**: 每 6 小时
- **条件**: 如果有待处理项目，询问用户是否开始处理
- **状态**: ⏳ 禁用（等任务系统完成后启用）
```
